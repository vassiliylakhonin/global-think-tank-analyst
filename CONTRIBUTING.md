# Contributing

Thanks for contributing.

## What is welcome

- Better prompts and output quality
- New real-world examples in `examples/`
- Clearer docs and adaptation guidance
- Bug fixes in skill formatting and structure

## How to contribute

1. Fork the repository
2. Create a branch: `feat/<short-name>` or `fix/<short-name>`
3. Make focused changes
4. Update docs/examples if behavior changes
5. Open a PR with:
   - Problem
   - Change
   - Before/after snippet
   - Risk/limitations

## Style expectations

- Keep outputs decision-useful and structured
- Mark assumptions and uncertainty explicitly
- Avoid deterministic claims under weak evidence
- Prefer concise changes over broad rewrites

## PR checklist

- [ ] Scope is clear and minimal
- [ ] README/SKILL docs updated if needed
- [ ] Examples added/updated
- [ ] No broken markdown/code blocks
